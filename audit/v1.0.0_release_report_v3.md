## Executive Summary

Aksu v1.0.0 ships a neural Turkish morphological atomizer (`"evlerinden" ŌåÆ "ev +Noun +PLU +POSS.3SG +ABL"`) built on a 5-seed BERTurk-based disambiguator achieving **98.32% exact-match** (std 0.11 pp, SLURM 5782508). The 80,537-entry TR-Gold-Morph v1 dataset is built, validated, and passes twine check. All 249 automated tests pass. The RC1 tag is cut locally.

Publication is blocked on four missing credentials: AWS S3 keys (DVC push), TestPyPI/PyPI tokens, HF_TOKEN, and ZENODO_TOKEN. Phase Q filed a halt report on 2026-05-16. Downstream phases R, S.2/S.3, T, U, V, and W cannot proceed until the user supplies those credentials. Three additional halt reports document measurement variance (BERTurk throughput), DualHead undertraining (deferred to v1.1), and TTC-3600 acquisition failure (deferred to v1.1). Seven audit claims are DEFERRED; all others are CONFIRMED.

**GitHub release notes paragraph:**

> Aksu v1.0.0 introduces a Python-native Turkish morphological atomizer replacing Zemberek. The 5-seed BERTurk ensemble achieves 98.3% exact-match on the 80,537-entry TR-Gold-Morph v1 dataset. The `aksu` package is installable from source (`pip install -e .`). PyPI, HuggingFace Hub, and Zenodo publication are pending credential setup; see `audit/v1.0.0_release_report_v3.md` for resume instructions.

**Claims remediated:** 28 of 35 audit claims confirmed; 7 deferred to v1.1.

## Reproducibility Statement

**What an external auditor CAN reproduce now from this repo:**

Clone `feat/aksu-readme-rewrite` at commit `d691f891cbae8b8b0fd3f2566ce2ac1b6b06976b`, install with `pip install -e ".[benchmark,train,data]"`, and run `make test` to verify all 249 tests pass. The BERTurk 5-seed ensemble evaluation can be reproduced by running `scripts/eval_all_models.py` with the bundled model weights (tracked via DVC; requires DVC pull, which in turn requires AWS credentials). Training wall-clock can be reproduced on any Xeon Scalable node with `make train-local` (CPU path only; BERTurk fine-tuning requires TRUBA akya-cuda). The build artifacts in `dist/` can be verified with `twine check dist/*`. The metrics snapshot at `audit/benchmark_results/metrics.json` (SHA-256: `39beea720d4b9a210366667aedf58ffb89e22231cbffeb1cb55c1b8e4b5c4ba2`) can be recomputed by running `scripts/ingest_metrics.py` after completing the SLURM jobs listed in the Numbers Table above.

**What is deferred and cannot be reproduced externally yet:**

DVC-tracked model weights and training data require AWS credentials for remote pull; without them, an auditor must retrain from scratch. The DualHead model's v1.1 GPU retrain has not been run; prior 84.7% EM figures should not be cited. TTC-3600 classification results are fully absent. PyPI, HuggingFace Hub, and Zenodo artifacts do not yet exist publicly, so a clean-VM install test is not possible. The OSCAR-based v2 dataset pipeline has not been staged.
